1. Package installation

Here we primarily use the rpart and partykit packages, while the others are used for data processing.

if (!require(rpart)) install.packages("rpart")
if (!require(partykit)) install.packages("partykit")
if (!require(caret)) install.packages("caret")
if (!require(rpart.plot)) install.packages("rpart.plot")

library(rpart)  #classification and regression trees
library(partykit)  #treeplots
library(caret)  #tune hyper-parameters
library(rpart.plot)

Data Reading and processing

The data can be downloaded at wbdc.data and wbdc.names from the following address: http://archive.ics.uci.edu/ml/machine-learning-databases/breast-cancer-wisconsin/. The data has been processed into panel data format.

Upon examining the data structure, we find that the first column is an ID column, which has no meaningful features and should be removed. The second column, “diagnosis,” is the response variable and is of character type. Generally, in R, for classification tasks, the response variable is required to be of factor type. Therefore, we need to convert its data type. The remaining columns are predictor variables and are of numeric type. The data dimensions are 97 samples with 9 features (including the response feature).

data(prostate)
prostate$gleason <- ifelse(prostate$gleason == 6, 0, 1)
sum(is.na(prostate))  ###missing value?
## [1] 0

Split the raw data into training and testing datasets.

library(sampling)
set.seed(123)

train_id <- strata(data, "diagnosis", size = rev(round(table(data$diagnosis) * 0.7)))$ID_unit
# traning data
train_data <- data[train_id, ]
# test data
test_data <- data[-train_id, ]

Regression trees

Here we are using the function rpart from the rpart package to complete the tutorial. First, we use the rpart function to build a classification tree model, and then plot the regression tree.

set.seed(123)
tree.pros <- rpart(lpsa ~ ., data = train_data)
length(tree.pros$frame$var[tree.pros$frame$var == "<leaf>"])  # 共有6个叶子
## [1] 6

To plot the complexity parameter table for the Rpart Fit, we select the one with the minimum RSS and the minimum xerror, where the split number is 5.

prp(tree.pros, type = 1, extra = 1, under = TRUE, split.font = 1, varlen = -10, box.col = ifelse(tree.pros$frame$var ==

Or one can use the rpart.plot package to visualize the regression tree.

rpart.plot(tree.pros)

Validation on test data.

party.tree.test <- predict(tree.pros, newdata = test_data)
rpart.resid <- party.tree.test - test_data$lpsa  #calculate residual
mean(rpart.resid^2)
## [1] 0.6136057

Prunning: prune the regression tree and obtain fewer branches.

cp <- min(tree.pros$cptable[5, ])
prune.tree.pros <- prune(tree.pros, cp = cp)
plot(as.party(prune.tree.pros))

Calculating RSS on test dataset.

party.pros.test <- predict(prune.tree.pros, newdata = test_data)

mean(rpart.resid^2)
## [1] 0.5267748

To prune the tree using the smallest cost complexity parameter (cp) value.

set.seed(123)
pruned.ct <- prune(cv.ct, cp = cv.ct$cptable[which.min(cv.ct$cptable[, "xerror"]),
    "CP"])
length(pruned.ct$frame$var[pruned.ct$frame$var == "<leaf>"])  
## [1] 5
plotcp(pruned.ct)

After pruning, plot the regression tree.

prp(pruned.ct, type = 1, extra = 1, split.font = 1, varlen = -10)

Prediction

party.prune.test <- predict(pruned.ct, newdata = test_data)
plot(as.party(pruned.ct))

mean(rpart.resid^2)
## [1] 0.5267748

ROC curves

### ROC
library(ROCR)
pred <- prediction(predictions = party.prune.test, labels = test_data$gleason)

perf <- performance(prediction.obj = pred, measure = "tpr", x.measure = "fpr")
perf
## A performance instance

    box.lty = 7, box.lwd = 2, box.col = "gray")
abline(a = 0, b = 1, lty = 2, col = "gray")
