2. Example data

The Stanford University Medical Center provided PSA (Prostate-Specific Antigen) data for 97 patients who underwent radical prostatectomy. Our goal is to build a predictive model using clinical test data to forecast post-operative PSA levels. The PSA level can be an effective prognostic indicator of the extent of recovery after surgery. Post-surgery, doctors will check patients’ PSA levels at various intervals and determine their recovery status using various formulas. A predictive model developed before surgery, combined with post-surgery data (not provided here), could enhance the diagnosis and treatment of prostate cancer and improve its prognosis.

Specifically, the dataset includes the following variables:

Here is the detailed description of the variables in the dataset:

  • lcavol: The logarithm of the tumor volume.
  • lweight: The logarithm of the prostate weight.
  • age: The age of the patient in years.
  • bph: The logarithm of the amount of benign prostatic hyperplasia (BPH), which refers to non-cancerous enlargement of the prostate.
  • svi: Seminal vesicle invasion, a binary indicator variable representing whether cancer cells have invaded the seminal vesicles beyond the prostate wall (1—Yes, 0—No).
  • lcp: The logarithm of capsular penetration, indicating the extent to which cancer cells have spread beyond the prostate capsule.
  • gleason: The patient’s Gleason score, given by a pathologist after a biopsy (range 2–10), indicating the degree of variation in cancer cells—the higher the score, the more aggressive the cancer.
  • pgg45: The percentage of Gleason scores that are 4 or 5.
  • lpsa: The logarithm of the PSA (Prostate-Specific Antigen) level, which is the response variable.
  • train: A logical vector for cancer status indicator.

3. Build a logistic regression model

We first build a logistic regression model using train as the dependent variable and all other variables as independent variables. We will construct the model and calculate the Variance Inflation Factor (VIF) and Generalized Variance Inflation Factor (GVIF) for linear, generalized linear, and other regression models to examine the linear relationships between the independent variables. We find that the VIF values are all greater than 2, indicating that the relationships between the variables are very tight, as shown below:

4. PCA

To address the issue of insignificant results, we need to perform dimensionality reduction by selecting the most important variables that significantly influence the dependent variable. We will use Principal Component Analysis (PCA) to obtain the principal components. We can create a scree plot to determine the number of principal components to retain. We will use Cattell’s scree test, which shows the relationship between eigenvalues and the number of principal components. The general principle is to retain principal components with eigenvalues greater than 1 and greater than the eigenvalues obtained from parallel analysis. We observe that three principal components meet this criterion, as shown below.

fa.parallel(prostate[, -10], fa = "pc", n.iter = 100, show.legend = FALSE, main = "Scree plot with parallel analysis")  #判断主成分个数
## Parallel analysis suggests that the number of factors =  NA  and the number of components =  2
abline(h = 1, lwd = 1, col = "green")

The principal() function in R can perform principal component analysis (PCA) based on the original data matrix or a correlation matrix. Here are the key arguments for the function:

  1. r: The correlation matrix or the original data matrix.
  2. nfactors: The number of principal components to extract (default is 1).
  3. rotate: The method of rotation to use (default is varimax).
  4. scores: Whether to calculate principal component scores (default is FALSE). Let’s use the principal() function to perform PCA on the prostate cancer dataset and identify the main components for each variable. In the example provided, we find that RC1 is mainly influenced by lpsa, RC2 by lbph, and RC3 by gleason.
pc <- principal(prostate[, -10], nfactors = 3, rotate = "none")

From the output, we can see that the first principal component has large loadings on five observed variables. The second principal component has large loadings on lweight and lbph. The third principal component has relatively smaller loadings, and the differences are not significant. The two principal components explain 73% of the variance in the data, but the results are not easily interpretable for the three principal components. Therefore, we consider performing principal component rotation to improve interpretability.

5. Rebuild logistic regression model

After completing the principal component analysis (PCA) and using the results to construct a logistic regression model with three principal components, we observed that the inflation factors are now around 1, indicating that multicollinearity has been effectively addressed. However, despite this improvement, the logistic regression model constructed with the principal components does not seem to be very suitable.

In comparison to the previous LASSO regression model, it appears that the latter performs better under these circumstances.

5. Outputs interpretation

The output results consist of two main parts:

Part 1: Loadings for PC1, PC2, and PC3 In this part, PC1, PC2, and PC3 represent the loadings of each observed variable on the two principal components. These loadings indicate the correlation between the observed variables and the principal components:

-h2: Represents the communality of the principal component, i.e., the proportion of variance explained by the principal component for each variable. -u2: Represents uniqueness, i.e., the proportion of variance in the variable that is not explained by the principal component. The sum of h2 and u2 equals 1.

Part 2: Description of the Two Principal Components In this part, the description of the two principal components includes:

  • SS loadings: Eigenvalues of the two principal components.
  • Proportion Var: Proportion of variance explained by each principal component, indicating the degree to which each principal component explains the data.
  • Cumulative Var: Cumulative proportion of variance explained by the principal components.

These descriptions provide insights into the variance explained by each principal component and their cumulative contribution to the total variance in the data.
