1. Package installation

Here, we mainly use the kknn and class packages, while the others are required for data processing.

Data Reading and processing

The data can be downloaded at wbdc.data and wbdc.names from the following address: http://archive.ics.uci.edu/ml/machine-learning-databases/breast-cancer-wisconsin/

The data has been processed into panel data format. Upon inspection of the data structure, it was found that the first column is an ID column with no meaningful features and thus needs to be removed. The second column, “diagnosis,” is the response variable and is categorical. In R, for classification tasks, response variables are typically required to be of factor type, so data type conversion is needed. The remaining columns are predictor variables and are of numeric type.

The data has dimensions of 568 samples and 32 features (including the response feature).

Split the raw data into training and testing datasets. The testing data will not be involved in training the model, and the optimal model parameters will be chosen based on the model’s performance on the testing data.

Typically, data splitting involves retaining 70% of the data for training and using 30% for testing. However, this ratio can be adjusted, but it’s generally recommended to have more training data than testing data to ensure sufficient learning of the model.

Additionally, in classification tasks, it’s important to ensure that the proportion of positive and negative samples is similar in both the training and testing datasets. This can be achieved through “stratified sampling”.

library(sampling)
set.seed(123)

train_id <- strata(data, "diagnosis", size = rev(round(table(data$diagnosis) * 0.7)))$ID_unit
# traning data
train_data <- data[train_id, ]
# test data
test_data <- data[-train_id, ]

KNN

In R, there are multiple packages available for KNN modeling. The caret package is commonly used for machine learning and integrates many popular algorithms. The parameter selection is as follows:

Use the trainControl function to set up 10-fold cross-validation and pass it to the trControl parameter. Use the preProcess parameter to set up standardization of the training data. Use the tuneLength parameter to set the number of values for k, and the model will perform parameter grid search tuning accordingly. The default evaluation metric for classification problems is accuracy.

Selecting K

Viewing the trained model, we can see that the accuracy of the models exceeds 95%, and the optimal value for k is 4.


library(caret)
## knn模型训练
control <- trainControl(method = "cv", number = 10)
grid1 <- expand.grid(.k = seq(2, 24, by = 1))
model <- train(diagnosis ~ ., train_data, method = "knn", trControl = control, preProcess = c("center",
    "scale"), tuneLength = 5, tuneGrid = grid1)

Weighted KNN

Sample imbalance can easily lead to incorrect results.

For example, if one class has a very large sample size while other classes have very small sample sizes, it may result in a new sample’s k neighbors being mostly from the large sample class.

Improvement methods:

Use weighted methods (where neighbors closer to the sample have higher weights) to improve the model. High computational cost: Since the distance from each test sample to all known samples needs to be calculated to find its k nearest neighbors, the computational load can be significant. Improvement methods:

Clip known sample points in advance, removing samples that do not significantly contribute to classification. This method is more suitable for class domains with larger sample sizes, while smaller sample size class domains may result in misclassification with this algorithm.

Weighted KNN algorithm: Use a Gaussian function to optimize the weights of samples at different distances. As the distance between the training sample and the test sample increases, the weight of that distance value decreases. Allocate greater weights to closer neighbors (the closer you are to me, the more similar I think you are to me, so I assign you a greater weight), while the weights of more distant neighbors decrease accordingly. Take the weighted average. We will use weighted KNN to optimize the model.
