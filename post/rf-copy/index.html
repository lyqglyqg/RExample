1. Package installation

Here we primarily use the randomForest package.

if (!require(randomForest)) install.packages("randomForest")

library(randomForest)

Data Reading and processing

The data can be downloaded from http://archive.ics.uci.edu/ml/machine-learning-databases/breast-cancer-wisconsin/. There are two files: wbdc.data and wbdc.names. The data has been processed into panel data format. Upon inspecting the data structure, the first column is an ID column with no meaningful features, so it needs to be deleted. The second column, “diagnosis,” is the response variable, which is categorical. In R, for classification tasks, the response variable is typically required to be of factor type, so it needs to be converted. The remaining columns are predictor variables, which are numeric. The dataset consists of 97 samples and 9 features (including the response feature).

BreastCancer <- read.csv("wisc_bc_data.csv", stringsAsFactors = FALSE)

Random forest

To build a classification random forest model for breast cancer diagnosis (malignant or benign), we’ll use the randomForest package in R. We’ll select the diagnosis (malignant or benign) as the response variable and construct the random forest model. We’ll then choose the model with the minimum RSS (Residual Sum of Squares).

set.seed(123)
rf.biop <- randomForest(diagnosis ~ ., data = train_data)
rf.biop

Optimal random number

Regression random forest

When building the regression result random forest model, we chose the pancreatic cancer data from the ElemStatLearn package. The response variable is continuous numerical, and we performed data reading and splitting.

library(ElemStatLearn)
data(prostate)
prostate$gleason <- ifelse(prostate$gleason == 6, 0, 1)
pros.train <- subset(prostate, train == TRUE)[, 1:9]
pros.test = subset(prostate, train == FALSE)[, 1:9]
