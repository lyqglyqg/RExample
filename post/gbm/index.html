Data Reading and processing

The data can be downloaded at wbdc.data and wbdc.names from the following address: http://archive.ics.uci.edu/ml/machine-learning-databases/breast-cancer-wisconsin/

The data has been processed into panel data format. Upon inspection of the data structure, it was found that the first column is an ID column with no meaningful features and thus needs to be removed. The second column, “diagnosis,” is the response variable and is categorical. In R, for classification tasks, response variables are typically required to be of factor type, so data type conversion is needed. The remaining columns are predictor variables and are of numeric type.

The data has dimensions of 568 samples and 32 features (including the response feature).

Data Preprocessing

Data preprocessing includes five steps. First, check if there is any missing data and count the number of missing values. Then proceed with the following steps:

  • Delete variables with low variance.
  • Remove variables that have a very high correlation with other independent variables.
  • Eliminate multicollinearity.
  • Standardize the data and impute missing values.
  • Feature selection using Recursive Feature Elimination (RFE).

Feature Selection

In data mining, not all independent variables are needed for modeling; instead, a subset of the most important variables is selected. This process is called feature selection. One algorithm for this is backward selection, which starts by including all variables in the model, then calculates their performance (such as error or prediction accuracy) and ranks the variables by importance. The most important variables are retained, and performance is recalculated, iterating this process to determine the optimal number of independent variables. One drawback of this algorithm is the potential for overfitting, which is why it should be coupled with an outer loop of sample partitioning.

The rfe function in the caret package can accomplish this task. The functions parameter determines the model used for ranking the independent variables, including:

  • rfFuncs (Random Forest)
  • lmFuncs (Linear Regression)
  • nbFuncs (Naive Bayes)
  • treebagFuncs (Bagged Trees)
  • caretFuncs (Custom Training Models) The method parameter specifies the sampling method, such as cross-validation (cv), bootstrapping (boot), and Leave-One-Out Cross-Validation (LOOCV).
ctrl = rfeControl(functions = caretFuncs, method = "repeatedcv", verbose = FALSE,
    returnResamp = "final")
BreastCancer$diagnosis = as.factor(BreastCancer$diagnosis)
Profile = rfe(BreastCancer[, -1], BreastCancer$diagnosis, rfeControl = ctrl)
print(Profile)

Visualizing Important Variables

We can use the featurePlot() function to visualize the range of values for each independent variable and compare different classifications.

For classification models, you can choose from: box, strip, density, pairs, or ellipse. For regression models, you can choose from: pairs or scatter.

featurePlot(x = trainData[, 2:21],

Define Testing Parameters

Before formal training, we first need to define the model training parameters using the trainControl function. The method parameter specifies the resampling method for cross-validation, number determines the number of folds, and repeats specifies the number of repetitions.

fitControl <- trainControl(
  method = 'cv',                   # k-fold cross validation
  number = 5,                      # number of folds
  savePredictions = 'final',       # saves predictions for optimal tuning parameter
  classProbs = T,                  # should class probabilities be returned
  summaryFunction=twoClassSummary  # results summary function
)
